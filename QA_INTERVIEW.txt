=== FITUR 4: Semantic Matcher (matcher.py) ===

[INI INTI AI-NYA]

[Apa ini?]
Menggunakan NLP untuk memahami makna CV dan Job Description, 
lalu hitung skor kecocokan 0-100%.

[Modelnya apa?]
all-MiniLM-L6-v2 dari Sentence Transformers.
Ukuran cuma 80MB, cepat, bisa jalan di laptop tanpa GPU.

[Gimana cara kerjanya?]
1. Job Desc di-encode -> vector angka (384 angka)
2. Teks CV di-encode -> vector angka (384 angka)
3. Hitung cosine similarity antara kedua vector
4. Hasil 0-1 dikali 100 -> skor persen

[Apa bedanya sama keyword matching biasa?]
Keyword matching: cari kata "machine learning" di teks
Semantic matching: paham bahwa "ML" = "machine learning" secara makna, 
meskipun kata-katanya beda. Jauh lebih akurat.

[Kenapa cosine similarity?]
Sederhana, cepat, standar untuk perbandingan vector embeddings.
Nilai -1 (berlawanan) sampai 1 (sama persis).

[Bisa ganti model?]
Bisa. Alternatif: all-mpnet-base-v2 (lebih akurat tapi lebih besar), 
atau multilingual models khusus Bahasa Indonesia.

[Kecepatan?]
Sangat cepat. encode 1 teks dalam milidetik.
Ratusan CV bisa diproses dalam hitungan menit.

Q: "Coba tunjukkan kode AI-nya bagaimana?"

@st.cache_resource
def get_model():
    from sentence_transformers import SentenceTransformer
    return SentenceTransformer('all-MiniLM-L6-v2')

def calculate_match_score(cv_text, job_desc_text):
    from sklearn.metrics.pairwise import cosine_similarity
    
    model = get_model()
    embeddings = model.encode([job_desc_text, cv_text])
    
    job_vector = embeddings[0].reshape(1, -1)
    cv_vector = embeddings[1].reshape(1, -1)
    
    score = cosine_similarity(job_vector, cv_vector)[0][0]
    return round(score * 100, 2)

def rank_candidates(cv_data, job_desc_text):
    results = []
    for cv in cv_data:
        score = calculate_match_score(cv["teks"], job_desc_text)
        reason = generate_reasoning(score)
        results.append({
            "Kandidat": cv["nama_file"],
            "Score_Kecocokan": score,
            "Alasan": reason
        })
    return sorted(results, key=lambda x: x["Score_Kecocokan"], reverse=True)

=== 3. Bias Mitigation & Anonimisasi (bias_mitigation.py) ===

Q: "Bagaimana kode untuk deteksi bias?"

def anonymize_text(text):
    # Hapus email
    text = re.sub(r'[\w\.-]+@[\w\.-]+\.\w+', '[EMAIL DIHAPUS]', text)
    # Hapus telepon Indonesia
    text = re.sub(r'\+62[\d\s\-\(\)]{8,15}', '[TELEPON DIHAPUS]', text)
    return text

def detect_potential_bias(cv_text):
    text_lower = cv_text.lower()
    warnings = []
    # Deteksi usia (pola "25 tahun", "lahir 1995", dll)
    for pat in AGE_INDICATORS:
        if re.search(pat, text_lower, re.IGNORECASE):
            warnings.append("Informasi usia terdeteksi (bisa memicu bias usia)")
            break
    # Deteksi suku/agama
    for pat in ETHNIC_INDICATORS:
        if re.search(pat, text_lower, re.IGNORECASE):
            warnings.append("Informasi suku/agama terdeteksi (bisa memicu bias)")
            break
    # Deteksi gender dari gaya bahasa
    male_count = sum(1 for w in GENDER_WORDS_MALE if w in text_lower)
    female_count = sum(1 for w in GENDER_WORDS_FEMALE if w in text_lower)
    if male_count > female_count + 2:
        warnings.append("Gaya bahasa cenderung maskulin")
    elif female_count > male_count + 2:
        warnings.append("Gaya bahasa cenderung feminin")
    return warnings

def score_fairness(gap_results_list):
    scores = [r["semantic_score"] for r in gap_results_list]
    mean_s = sum(scores) / len(scores)
    std_dev = (sum((s - mean_s)**2 for s in scores) / len(scores))**0.5
    return {
        "rata_rata_skor": round(mean_s, 2),
        "standar_deviasi": round(std_dev, 2),
        "distribusi_wajar": std_dev < 30,
    }

[Penjelasan untuk interview:]
- Anonimisasi: hapus PII (Personal Identifiable Information) 
  sebelum CV diproses. Ini langkah pertama mitigasi bias.
- Deteksi usia: cari pola "25 tahun", "born 1995", "tanggal lahir"
- Fairness score: kalau distribusi skor terlalu lebar (std > 30),
  ada indikasi sistem bias ke kelompok tertentu.
- Ini masih rule-based, belum ML. Ke depannya bisa pake model 
  fairness detection yg lebih canggih.

# Extract skills dari 70+ keywords
SKILL_KEYWORDS = [
    "python", "java", "sql", "docker", "machine learning", ...
]

def extract_skills(text):
    text_lower = text.lower()
    found = []
    for skill in SKILL_KEYWORDS:
        pattern = r'\b' + re.escape(skill) + r'\b'
        if re.search(pattern, text_lower):
            found.append(skill.title())
    return list(dict.fromkeys(found))  # unique, urut sesuai muncul

def extract_experience_years(text):
    patterns = [
        r'(\d+)[\+]?\s*(?:tahun|years|thn|yr)\s*(?:pengalaman|experience)',
        r'(?:pengalaman|experience)[^\d]{0,30}(\d+)[\+]?\s*(?:tahun|years|thn|yr)',
        r'(\d+)[\+]?\s*(?:tahun|years|thn|yr)',
    ]
    for pat in patterns:
        match = re.search(pat, text.lower())
        if match:
            return int(match.group(1))
    return None

[Penjelasan untuk interview:]
- Skill matching: cocokin 70+ keywords skill ke teks CV
- Regex fleksibel: handle "5 tahun pengalaman" dan "pengalaman 5 tahun"
- Ini masih rule-based. Ke depannya bisa pake NER model biar
  lebih akurat untuk CV yang formatnya unik.

def extract_text_from_file(uploaded_file):
    text = ""
    with pdfplumber.open(uploaded_file) as pdf:
        for page in pdf.pages:
            extracted = page.extract_text()
            if extracted:
                text += extracted + "\n"
    return text

[Penjelasan untuk interview:]
